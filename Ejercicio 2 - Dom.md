# Ejercicio 2 - ***DOM***

## **Introduccion**
  DOM (Modelo de Objetos del Documento) es un concepto importante en el desarrollo web. Funciona como un puente entre los documentos HTML y los lenguajes de programación, permitiendo a los usuarios interactuar y modificar las páginas web. El DOM proporciona una forma de acceder a elementos, atributos y texto. Esto hace posible actualizar contenido, responder a acciones de los usuarios y crear experiencias interactivas en la web.

## **Síntesis**
  El DOM es una API que permite a los programas leer y modificar documentos. Cuando un navegador carga un archivo HTML, genera un árbol de tres nodos. En la parte superior del árbol está el documento, que funciona como el punto de entrada para acceder a todos los demás nodos. Por ejemplo, el elemento HTML contiene un head y un body, y cada uno de esos se ramifica en otros elementos. Esta estructura permite a los usuarios recorrer el documento de manera organizada.
  Aunque JavaScript es el lenguaje más común para manipular el DOM, el modelo en sí puede ser accedido por otros lenguajes como Python. Esta independencia asegura que el DOM siga siendo una interfaz consistente en diferentes entornos. Los desarrolladores pueden usar métodos para localizar elementos específicos y luego modificar sus propiedades, estilos o contenido.El DOM proporciona acceso a atributos como nodeList o NamedNodeMap, que permiten a los usuarios trabajar con grupos de elementos o atributos de manera eficiente.
  El documento representa todo el documento HTML o XML. Es la parte superior del árbol DOM y proporciona acceso a todos los elementos dentro de la página web. El elemento se refiere a etiquetas HTML individuales como div, p o h1. Los elementos son nodos que se pueden crear, modificar o eliminar. NodeList es una colección de nodos, como los que se devuelven por métodos como getElementsByTagName().Un atributo representa las características de los elementos, como el color y la forma.NamedNodeMap es una colección de atributos a los que se puede acceder por nombre o por índice. Es útil para manejar varios atributos de un elemento.

  ## **Reflection**
  Lo que encuentro útil del DOM es cómo permite a los usuarios interactuar con las páginas web. Sin el DOM, los documentos HTML permanecerían estáticos y no interactivos, o no tanto. Al aprender a cambiar y mover nodos, los usuarios pueden crear páginas que respondan a otros usuarios en tiempo real. Esto no solo mejora la forma en que la gente lo usa, sino que también mejora la experiencia. Me gusta cómo el DOM hace que sea más fácil entender y trabajar con páginas web complejas, por la manera en que organiza todo.

  ## **Conclusion**
  El DOM permite a los usuarios transformar documentos simples en aplicaciones interactivas. Al proporcionar una estructura de árbol clara, diferentes tipos de datos y control sobre cómo se muestra y actualiza el contenido. El DOM no solo es un fundamento importante, sino también una herramienta práctica que anima a los desarrolladores a diseñar experiencias web receptivas, atractivas y modernas.
